Explenations for independent extension

document.addEventListener("keydown", disableF5):

This event catches all keydown events and checks for F5 (refresh operation) "keydown" event.
after such key code was identified (key-code 116), the event calls a function named "disableF5()" which is in Worker.js.

*this event catches all keydown events that happen from the keyboard and mouse.
-------------------------------------------------------------------------------------------

disableF5(event):

disableF5(): this function will verify that the key pressed is actually F5 and disables it.
after verification that F5 was pressed, it terminates the F5 operation and calls refreshWaiter().

*this function will do nothing if the key code is NOT 116 (F5)
*this function will not catch any other kind of refresh operation like Ctrl+R nor manual refresh.
-------------------------------------------------------------------------------------------

refreshWaiter():

This function will hide all menus and the board, and show the player a decision "menu" with two buttons
to decide weather to refresh or not.
-if the user chose to refresh- the local storage will be cleaned,
all statistics will be reseted and you will go to the main menu to start a new game
-if the user chose NOT to refresh- the game board will be shown again (with no game progress nor player statistics changes -game will continue as nothing happend )
and all other forms and menus will be hidden
-------------------------------------------------------------------------------------------
